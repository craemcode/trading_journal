# trading_journal
A simple trading journal for personal use. 

Trading journals allow one to track their trading journey. One can record what pairs they traded, what they risked, and their outcome.
The journal should also show summary data such as average Risk-to-reward and win rate. 

## Funtional Requirements.

The application allows a user to enter the following details.
<ol>
  <li> Starting capital</li>
  <li> Evaluation cycle (no of trades to draw stats) </li>
</ol>

Then, for each trade, the user should provide:
<ul>
  <li> Symbol Name</li>
  <li> Direction bias (long or short)</li>
  <li> Reason for taking the trade</li>
  <li> Risk:Reward Ratio</li>
  <li> Outcome (win or loss)</li>
  <li> Comments about the outcome</li>
</ul>

Over time, the UI should show how the balance has grown or depleted. 

## Non functional requirements

The application is limited to the following non-functional requirements.
<ol>
  <li> Support both mobile and desktop interface (responsive design)</li>
  <li> Work on the browser (web application)</li>
  <li> Authentiction to separate users' data </li>
</ol>
